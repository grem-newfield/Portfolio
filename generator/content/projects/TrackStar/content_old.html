TrackStar

Student Project: Racing Game made with Godot 4.4

Project Overview

As part of a student group project, I contributed to the development of a racing game built in the Godot Engine. I proposed to clone a Ridge Racer type game,so focus on drifting and nature scenery. This means unique arcadey physics and drifting. Unlike a racing simulator the drifting in this game shouldnt decrease your speed, but instead increase it to help with overtakes of the AI cars around corners.

My Role and Contributions

I took on the roles of Code Lead, Level Designer, and Level Artist, I took on a multifaceted role, both technical implementation and creative design. Below, I reflect on my contributions, the challenges faced, and the skills I honed during this work.

Code Leadership and System Architecture

Though not formally appointed as the project lead, I de facto stepped into a leadership role due to my experience with Godot, game design, and code architecture. I delegated tasks to team members based on their strengths, hoping for efficient progress while maintaining a cohesive vision.

My primary coding contributions included designing a Manager System to handle the game’s core functionalities:
  • UI Manager: Handles scene transitions for menus and race states.
  • Tour Manager: Loads game maps dynamically.
  • Player Manager: Tracks player state (e.g., name, score, laps, timers).
  • Save Manager: Save/load with data validation.
  • Logger Class: Streamlined debugging.

Physics-Based Car Simulation

The fun part of the project for me was implementing a car using a RigidBody and raycast-based wheels. This system mimicked real-world vehicle dynamics, particularly for a sports/drift car.

Key features:

    Spring Force and Suspension: Raycasts simulated wheel suspension, providing realistic terrain interaction.

    Sideways Slip and Drifting: Configured forces to enable controlled drifting, with only the rear wheels applying forward velocity to emulate a rear-wheel-drive sports car.

    The cars forces are moderated by curves:

    Top Speed Falloff: Prevented indefinite acceleration.

    Steering Speed Falloff: Reduced steering sensitivity at high speeds for realism.

    Tyre Slip Curve: Enabled drifting at higher speeds by adjusting grip dynamically.

    Lerp Camera: Implemented a smooth, interpolated camera system that followed the car, enhancing the player’s immersion.

Level Design and Art

As a Level Designer and Level Artist, I crafted the game’s environment using the Terrain3D plugin in Godot. This plugin allowed me to sculpt a heightmap with brushes and apply textures to create a visually rich racing track. I also focused on establishing a realistic atmosphere through careful lighting, fog, and environmental effects, immersing players in the racing experience.

In Blender, I modeled game assets (e.g., obstacles, props) and colliders to ensure accurate physics interactions. Additionally, I developed a custom workflow to enhance AI pathfinding: Blender Plugin: Created a plugin that exports Bezier curves as JSON data, defining AI navigation paths.

Godot Plugin: Built a complementary plugin to import JSON data and generate curve nodes in Godot, enabling smooth AI movement along the track.
